# .agent.md

## Agent: LocalAI Intent -> Software

**Role**: Build working software locally from a natural-language prompt using Gemma 4 via Ollama, with a Tauri UI as the demo surface.

## Core architecture (stateless worker)

Every step is the same pattern:

`read input file -> call Gemma once -> write output file -> exit`

No hidden memory across steps. The filesystem is the state.

## Ollama configuration

- **URL:** `http://localhost:11434/api/chat`
- **Model:** `gemma4:26b-optimized`
- **Sampling:** `temperature=1.0, top_p=0.95, top_k=64`

## Pipeline overview (engine)

`intent-to-software/user_prompt.txt`
-> `step1_compress.py` -> `compressed_intent.txt`
-> `step2_mockui.py` -> `mock_ui.html`
-> `step3_parse.py` -> `features.json`
-> `step4_dag.py` -> `dag.json`
-> `step5_tasks.py` -> `tasks.json` + `tests.json`
-> `step6_build.py` -> `output/` (generated software)

## Step 6: build loop (per-task, per-test)

Step 6 runs **one task at a time**:

- ask Gemma for the single target file
- write that file into `output/`
- run that task’s test (from `tests.json`)
- retry on failure, then move to the next task

It also writes:
- `output/PROJECT_CONTEXT.md` (rolling context tail)
- `output/build_manifest.json` (what was generated)

## UI (submission surface)

The Tauri UI is `step7_ui/`.

- Requires GitHub Device Flow so exports go to the **user's** GitHub account
- Shows exact step stderr/stdout on error and saves `intent-to-software/ui_last_error.txt`
